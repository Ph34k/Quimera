## 2024-05-24 - SSRF in Agents API
**Vulnerability:** The `IScoutAgent` and `IExecutionAgent` were performing HTTP requests using target URLs specified in the payload without any validation, enabling Server-Side Request Forgery (SSRF). Attackers could use the agents to access internal or loopback IP addresses. Additionally, `IExecutionAgent` followed redirects, making it possible to bypass basic URL checks if an external server redirected to an internal IP.
**Learning:** Even simple web scraping or HTTP agents can be abused to attack internal networks if target endpoints aren't carefully vetted before initiating connections. The redirect functionality of the HTTP library can undermine initial validation checks if redirects themselves aren't also inspected.
**Prevention:** Always validate target URLs before issuing requests by resolving their DNS and verifying the resulting IPs are public (not private, loopback, or link-local). If the client must follow redirects, inject an event hook (e.g., `event_hooks` in `httpx`) to inspect the `Location` header and ensure the new destination also adheres to the safety criteria before following it.
