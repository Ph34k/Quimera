## 2024-05-20 - Fix SSRF vulnerabilities in Agents
**Vulnerability:** The codebase allowed making requests to arbitrary URLs provided by users via `target_url` in both `IScoutAgent` and `IExecutionAgent` without validation. `IExecutionAgent` also followed redirects blindly (`follow_redirects=True`).
**Learning:** This architectural flaw permitted unauthenticated users to make the server perform HTTP requests to any address, creating a Server-Side Request Forgery (SSRF) vulnerability. This is exceptionally dangerous if the backend is hosted on internal networks (e.g., AWS, GCP, Azure), exposing metadata endpoints or internal APIs.
**Prevention:** Always validate all user-provided URLs using an internal safety check like `is_safe_url` that explicitly resolves the URL to its underlying IP(s) and ensures it is a public IP, thus rejecting private, loopback, or link-local addresses. Always enforce this validation on redirect `Location` headers if `follow_redirects` is required.
