## 2024-05-24 - [CRITICAL] Prevent SSRF in Agent HTTP Requests
**Vulnerability:** Agents (`IScoutAgent` and `IExecutionAgent`) accepted raw URLs and immediately performed `httpx.get` on them without validation. This allowed an attacker to supply internal IPs, resulting in Server-Side Request Forgery (SSRF). Additionally, `follow_redirects` was not explicitly configured, allowing potential bypass via an attacker-controlled external server redirecting to an internal IP.
**Learning:** `IExecutionAgent` explicitly requires `follow_redirects=True` for its stealth web driving feature. In these functional exception cases, validating the initial URL via `is_safe_url` remains essential, even if follow_redirects must remain True to fulfill functional requirements.
**Prevention:** Implement and enforce a comprehensive `is_safe_url` utility function that blocks requests to private, loopback, link-local, and unspecified IP ranges. Explicitly pass `follow_redirects=False` in HTTP clients unless functionality fundamentally depends on redirects. For `IExecutionAgent`, the initial URL must be validated despite the redirect requirement.
