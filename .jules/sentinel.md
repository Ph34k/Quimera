## 2024-05-24 - SSRF in HTTP Agents via `httpx.get`
**Vulnerability:** Found critical Server-Side Request Forgery (SSRF) in `app/domain/agents.py` where `IScoutAgent` and `IExecutionAgent` make HTTP requests to user-provided `target_url` without any validation against internal/private IP ranges.
**Learning:** Basic SSRF protections often miss IPv6 formats (e.g. `[::1]`) and are susceptible to DNS rebinding (TOCTOU). To properly secure requests, validation must occur at the socket layer resolving the exact IP used for the request, or by validating the hostname immediately before connection, accounting for all IPv4 and IPv6 private/loopback/link-local/unspecified addresses.
**Prevention:** Implement an `is_safe_url` function that parses the URL, resolves the hostname using `socket.getaddrinfo`, and checks the resulting IP addresses against `ipaddress` rules (checking for `.is_private`, `.is_loopback`, `.is_link_local`, `.is_unspecified`). Apply this validation before making any HTTP request using user input, and ensure `follow_redirects=False` unless strictly necessary (like in `IExecutionAgent`).
