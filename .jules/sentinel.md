## 2024-04-20 - Prevent SSRF in Agent HTTP Requests
**Vulnerability:** The agents (`IScoutAgent`, `IExecutionAgent`) accepted arbitrary user-provided URLs in their payloads and performed an HTTP request using `httpx` without any validation. This exposed the application to Server-Side Request Forgery (SSRF), allowing attackers to potentially scan or access internal resources, private IPs, or the loopback interface on the host machine.
**Learning:** External user inputs passed as URLs directly to HTTP clients must be explicitly validated. Allowing unfettered HTTP requests by default is dangerous. Additionally, managing redirects securely is critical—disabling redirects where not strictly necessary (`follow_redirects=False`) minimizes the attack surface.
**Prevention:** Implement and enforce a strict validation function (`is_safe_url`) that parses the URL and resolves its hostname. Reject any request if the resolved IP maps to a private, loopback, link-local, or unspecified address space using Python's `ipaddress` library before making the HTTP call. Disable automatic redirects whenever feasible.
